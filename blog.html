var, let and const

var is not block scop and it is hoisted. Means we don't need to declare before use. Also var is global so that we can access it from any where in the current code. Sometime it can make some issue so we will never use var.

On the other hand let and const introduce in es6, let and const are block scop. It must be declared before use or else you will get an error. let can be re-declared but you re-declaring const is impossible. And can't be accessible globally. So there is no chance you declare same variable again and again. This will make you code more bug free. And we will always use let and const instead of var.

Use cases of null and undefined

null and undefined are both used to represent if the value of a data is not found or empty. However, there are some differences between them. undefined is used when a variable has been declared but has not been assigned a value. On the other hand, null is used to represent an intentional absence of any object value.

undefined is a default value for function arguments and return values. It can also be used to check if a variable has been initialized or not. But null is used to indicate that an object does not exist or that it has no value.

What do you mean by REST API?

A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style. It is a set of rules that define how applications or devices can connect to and communicate with each other. REST APIs provide a flexible, lightweight way to integrate applications and to connect components in microservices architectures.
